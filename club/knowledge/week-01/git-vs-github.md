# Git은 엔진, GitHub는 도로

> 둘을 구분하는 순간 혼란의 절반이 사라집니다.

## 구분

| | **Git** | **GitHub** |
|---|---|---|
| 무엇 | 버전 관리 시스템 (2005) | Git 저장소를 올려두는 클라우드 서비스 (2008) |
| 어디서 | **내 컴퓨터** | **클라우드** |
| 누구와 | 나 혼자 | 팀 전체 — 협업·공유·자동화 |
| 제공 | commit, branch 등 이력 관리 | Issues, PR, Actions 등 협업 도구 |

비유: Git이 자동차 엔진이라면 GitHub는 도로망입니다. 엔진만 있어도 굴러가지만(로컬 버전 관리), 도로에 올라가야 다른 사람과 만납니다(공유·협업). 전 세계 1억 명 이상이 이 도로를 쓰고 있고 `[2차]`, 이 프로그램의 커리큘럼 레포 자체도 GitHub 위에서 운영됩니다.

## 이 수업에서는

- **Git**: 내 워크스페이스의 판단 이력을 세이브 (매 수업)
- **GitHub**: `/submit`으로 제출할 때 내 작업이 올라가는 곳
