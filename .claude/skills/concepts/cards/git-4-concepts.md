# 비개발자를 위한 Git 4개념 — 집 · 세이브 · 레이어 · 리뷰요청

> 이 4개 비유만으로 첫날 제출까지 갈 수 있습니다. 명령어는 외울 필요 없습니다 — AI에게 시키면 됩니다.

## 4개념

| 개념 | 비유 | 뜻 |
|---|---|---|
| **Repository (repo)** | 프로젝트의 집 | Google Drive 폴더 + 모든 변경이력이 자동 기록되는 곳 |
| **Commit** | 이유가 적힌 세이브 포인트 | 게임 세이브인데 "왜 세이브했는지"도 함께 기록 |
| **Branch** | 포토샵 레이어 | 원본은 보존하고, 수정은 분리된 층에서 |
| **Pull Request (PR)** | "이렇게 바꿔봤는데 봐줘" | 변경 이유 + 피드백 + 반영으로 이어지는 리뷰 요청서 |

투자자에게 이것이 중요한 이유: **판단의 변경 이력이 남기 때문**입니다. "언제, 무엇을 근거로, 왜 생각을 바꿨는가"가 커밋 로그에 쌓이면, 내 판단 과정 자체가 검토 가능한 자산이 됩니다. `제안서_최종_진짜최종_v2.pptx` 방식으로는 절대 얻을 수 없는 것입니다.

## 근거

이전 기수 실측 — 이 비유 세트로 배운 비개발자 참가자의 46%가 당일 Git/GitHub 과제를 수행했습니다.

## 직접 해보기

`내 워크스페이스(students/{내 ID}/)의 지금 상태를 commit 해줘. 커밋 메시지에 "왜"가 들어가게 써줘` — 끝나면 `방금 커밋에 뭐가 기록됐는지 보여줘`로 눈으로 확인하세요.
